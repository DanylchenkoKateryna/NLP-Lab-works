## 3. Cleaning & Normalization Rules Added

| Step | Rule | Detail |
|---|---|---|
| `clean_text` | Remove quote lines | Strip lines beginning with `>` (cited replies) |
| `clean_text` | Remove newsgroup headers | `From:`, `Subject:`, `Organization:`, etc. |
| `clean_text` | Remove `In article <…>` refs | Citation artifacts common in newsgroups |
| `clean_text` | Remove signatures | Block after `--` delimiter |
| `clean_text` | Decode HTML entities | `&amp;` → `&`, `&lt;` → `<`, etc. |
| `clean_text` | Collapse whitespace | Multiple spaces/tabs → single space |
| `normalize_text` | Curly → straight apostrophes | `'` `'` → `'` |
| `normalize_text` | Curly → straight quotes | `"` `"` → `"` |
| `normalize_text` | Em/en-dash normalization | `—` → `--`, `–` → `-` |
| `normalize_text` | Non-breaking space | `\u00a0` → ` ` |
| `normalize_text` | Cyrillic homoglyphs | Latin-looking Cyrillic → Latin |
| `mask_pii` | Mask URLs | → `<URL>` |
| `mask_pii` | Mask emails | → `<EMAIL>` |
| `mask_pii` | Mask phones | → `<PHONE>` |
| `sentence_split` | NLTK Punkt (English) | With number/version protection |

## 4. Five Most Painful Edge Cases

| # | Edge Case | Problem | Solution |
|---|---|---|---|
| 1 | **Quote cascades** (`>>>` nested replies) | Layers of citation pollute the content | `_QUOTE_LINE` regex strips all `^>+.*` lines |
| 2 | **Email addresses in body text** | 5,306 raw occurrences — would leak author identity | Masked to `<EMAIL>` before any ML use |
| 3 | **Version numbers (`1.2.3`)** | NLTK incorrectly splits sentences at dots in versions | Pre-masked with `__VER__` placeholder before splitting |
| 4 | **Curly apostrophes / quotes** | Unicode variants break string matching and tokenizers | Normalized to ASCII equivalents in `normalize_text` |
| 5 | **Signature blocks** | Author name, email, phone in every message tail | Stripped by `--` signature delimiter detection |
