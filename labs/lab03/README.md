## Stanza Configuration

```
processors : tokenize, pos, lemma
language   : en
use_gpu    : False
max_words  : 300  (text truncated before Stanza for performance)
```

PII tokens (`<URL>`, `<EMAIL>`, `<PHONE>`) are protected before Stanza via
placeholder substitution (`urltoken`, `emailtoken`, `phonetoken`) and restored
after lemmatization.

## Lemmatization Edge Cases

Edge cases are defined in `tests/ling_edge_cases.jsonl` (20 cases).

| Category | Example issue |
|---|---|
| `pii_token` | `<URL>` / `<EMAIL>` must survive unchanged |
| `abbreviation` | BTW, IMHO, RAM, ROM — wrong POS/lemma |
| `contraction` | "don't" → "do" + "not" |
| `hyphenated` | "follow-up" split into multiple tokens |
| `possessive` | "God's" → lemma "god" + PART "'s" |
| `archaic` | "thou shalt" — archaic morphology |
| `proper_noun` | "Jesus Christ", "alt.atheism" |
| `internet_slang` | AFAIK, IMHO — tagged as NOUN/PROPN |
| `number_unit` | 66MHz, 3.14 GHz — garbled lemmas |
| `technical_term` | "electrolytic capacitor", "data" (datum?) |
